# 流星运行时 — 设计文档

> 流星 · 流体语言通用执行 (Flux Runtime — Chinese First)
> 版本 0.1.0

---

## 一、设计哲学

### 1.1 为什么是「中文优先」？

流星运行时不是英文 flux-runtime 的翻译版。我们重新设计了自然语言解释层，
让中文语法结构从根本上塑造系统架构。这意味着：

- **主题-评论 (Topic-Comment)** 是默认句法结构，而非主语-谓语
- **量词 (Classifier)** 系统构成了类型层次的基础
- **零形回指 (Zero Anaphora)** 实现了隐式参数传递
- **词汇即类型系统** — 中文的词汇分类直接映射到程序类型

### 1.2 主题-评论 vs 主语-谓语

英文和大多数印欧语言采用**主语-谓语 (Subject-Predicate)** 结构：

```
Subject + Verb + Object
"The calculator computes 3 + 4"
```

中文天然倾向**主题-评论 (Topic-Comment)** 结构：

```
主题 + 评论
"这个计算，结果是七" (This computation, result is seven)
"三加四，算一下" (Three plus four, compute it)
```

在流星运行时中，这种结构体现在：

1. **R63 主题寄存器** — 每次计算的主题自动存入 R63，后续语句可以引用
2. **零形回指** — 省略主语时自动使用主题寄存器的值（"再加三" → 隐式 "主题+3"）
3. **模式匹配优先级** — 中文模式优先匹配主题-评论句型

### 1.3 量词即类型系统

中文量词系统是世界上最精细的名词分类系统之一。每个量词约束了一类名词：

| 量词 | 类型 | 可搭配名词 |
|------|------|-----------|
| 艘 | 船只 | 船、舰、艇 |
| 台 | 机器/设备 | 计算机、发动机、雷达 |
| 本 | 书籍/文件 | 书、册、日志 |
| 条 | 长形事物 | 河、消息、指令 |
| 位 | 人（敬称） | 船长、工程师 |
| 次 | 次数 | 循环、计算、尝试 |

在流星运行时中，量词不仅是语法元素，更是**编译时类型检查**的基础：

```python
# 正确：量词与名词匹配
"创建 三 艘船只"  ✅  艘 → 船只类型

# 错误：量词与名词不匹配
"创建 三 本船只"  ❌  本 → 书籍类型 ≠ 船只
```

### 1.4 零形回指 (Zero Anaphora)

中文允许省略上下文中已明确的成分，这是**零形回指**现象：

```
加载 寄存器零 为 十       # 主题 = 10
再加三                    # 零形回指：10 + 3 = 13
然后 乘 二                # 零形回指：13 × 2 = 26
```

在英文中，你必须显式写出主语：
```
Load R0 as 10
Add R0 and 3
Then multiply by 2
```

零形回指在流星运行时中通过**主题寄存器 (R63)** 实现：
- 每次操作的结果自动更新 R63
- 省略参数时自动引用 R63
- 代词 "它" 和 "其" 也指向 R63

## 二、编译管道

```
中文自然语言输入
       │
       ▼
  ┌──────────────┐
  │  主题-评论解析  │ ← 提取主题，建立上下文
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  量词验证     │ ← 检查量词-名词匹配
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  模式匹配     │ ← 中文NL模式 → 捕获组
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  值解析       │ ← 中文数字 → 阿拉伯数字
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  汇编展开     │ ← 模板 + 值 → 汇编文本
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  字节码编码   │ ← 汇编 → 字节数组
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  VM 执行     │ ← 字节数组 → 计算结果
  └──────────────┘
```

## 三、架构层次

### 3.1 语言无关层（与 flux-runtime 主运行时完全一致）

| 层 | 组件 | 说明 |
|----|------|------|
| L1 | Op 操作码集 | 64个操作码，变长编码(1-8字节) |
| L2 | 编码器 (Encoder) | 汇编文本 → 字节数组 |
| L3 | 反汇编器 (Decoder) | 字节数组 → 可读汇编 |
| L4 | 虚拟机 (VM) | 64寄存器栈式执行引擎 |
| L5 | FIR (中间表示) | 未来扩展：优化和变换 |

### 3.2 中文特定层

| 层 | 组件 | 说明 |
|----|------|------|
| L6 | 中文数字解析 | 中文数字 ↔ 阿拉伯数字 |
| L7 | 量词系统 | 量词验证与类型推断 |
| L8 | NL模式定义 | 中文正则模式 → 汇编模板 |
| L9 | 中文解释器 | 模式匹配 + 编译 + 执行 |
| L10 | CLI界面 | 中文命令行交互 |

### 3.3 主题寄存器约定

| 寄存器 | 名称 | 用途 |
|--------|------|------|
| R0 | 结果寄存器 | 操作结果默认存入 |
| R1-R9 | 天干寄存器 | 甲(R0)乙(R1)丙(R2)... |
| R10 | 寄存器十 | 扩展通用寄存器 |
| R63 | 主题寄存器 | 零形回指上下文 |

## 四、中文NL模式示例

### 4.1 算术运算

```
计算 三加四        → MOVI R0, 3; MOVI R1, 4; IADD R0, R0, R1  → R0=7
计算 五乘六        → MOVI R0, 5; MOVI R1, 6; IMUL R0, R0, R1  → R0=30
五的阶乘           → (循环) IMUL R1, R1, R2; DEC R2; JNZ      → R0=120
```

### 4.2 寄存器操作

```
加载 寄存器零 为 四十二  → MOVI R0, 42
减少 寄存器零            → DEC R0
```

### 4.3 智能体通信

```
告诉 甲 报告船位       → TELL R0, R1
询问 乙 当前船速       → ASK R0, R1
广播 全体返航          → BROADCAST R0, R1
```

## 五、与主运行时的关系

```
flux-runtime (英文/语言无关)
├── opcodes.ts          → Op 操作码  ═══ identical ═══  interpreter.py Op
├── encoder.ts          → 编码器      ═══ identical ═══  interpreter.py encode_assembly
├── decoder.ts          → 反汇编器    ═══ identical ═══  interpreter.py disassemble
├── vm.ts               → 虚拟机      ═══ identical ═══  interpreter.py FluxVM
│
├── vocabulary.ts       → 英文NL解释  ─── NOT used ───
│
flux-runtime-zho (中文优先)
├── interpreter.py L1-L5  → 语言无关层 (Python移植)
├── interpreter.py L6-L10 → 中文特定层
├── vocabulary/math.ese    → 中文数学词汇
├── vocabulary/maritime.ese → 中文航海词汇
├── vocabulary/a2a.ese     → 中文A2A词汇
└── vocabulary/classifiers.toml → 量词类型系统
```

**关键设计决策**：语言无关层（L1-L5）保持与主运行时二进制兼容。
任何用中文编译出的字节码，都可以在英文版VM上执行，反之亦然。
中文特色仅在编译前端（L6-L10），执行后端完全共享。

## 六、未来扩展

1. **FIR (Flux Intermediate Representation)** — 添加优化层
2. **瓦片系统 (Tile System)** — 支持更复杂的控制流
3. **四声解析** — 使用声调消歧同音词
4. **繁体支持** — 繁简体双向转换
5. **方言层** — 粤语、闽南语等方言NL模式
6. **LLM集成** — 用大语言模型处理未覆盖的模式
